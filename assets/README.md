# Assets Directory

This directory is for storing images and other media assets for your GitHub profile.

## üìÅ Recommended Assets

### Header Banner (Optional)
If you want to create a custom static banner instead of using the dynamic one:
- **Filename:** `header.png` or `header.jpg`
- **Recommended size:** 1920x320 pixels
- **Format:** PNG or JPG
- **Tools to create:**
  - [Canva](https://www.canva.com/) - Free online design tool
  - [Figma](https://www.figma.com/) - Professional design tool
  - [Photopea](https://www.photopea.com/) - Free Photoshop alternative

### Profile Picture
- **Filename:** `profile.png`
- **Recommended size:** 500x500 pixels (square)
- **Format:** PNG with transparent background preferred

### Project Screenshots
For showcasing projects:
- Create a folder for each project (e.g., `mercadolibre/`, `globant/`)
- Use descriptive names (e.g., `dashboard-screenshot.png`)
- Optimize images for web (compress to reduce file size)

### Tech Stack Icons
For custom technology icons or logos:
- **Folder:** `icons/`
- **Size:** 64x64 pixels recommended
- **Format:** PNG or SVG

## üé® Using Custom Images in README

To use images from this folder in your README:

```markdown
![Alt text](./assets/header.png)
```

Or with HTML for more control:
```html
<img src="./assets/header.png" alt="Header" width="100%" />
```

## üí° Tips

1. **Optimize images** before uploading (use tools like TinyPNG or ImageOptim)
2. **Use descriptive filenames** (e.g., `mercadolibre-dashboard.png`)
3. **Keep images under 1MB** each for faster loading
4. **Use PNG** for images with transparency, **JPG** for photos
5. **Consider dark mode** - use images that look good on both light and dark backgrounds
